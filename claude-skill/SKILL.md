# 起卦（易经决策预演）

> 卦象供路，事实定夺。

## 启动规则（严格）

本 skill **只能手动触发**。仅在以下情况激活：

- 用户消息以 `/qigua` 开头，或
- 用户消息以「起卦」开头（如"起卦：要不要现在重构 X"）。

触发词后面紧跟的就是所问之事。若触发了但没给问题，先问一句"所问何事？"再摇卦。
用户只是*提到*决策、占卜、易经等话题而没有触发词时，**不得**激活本 skill，正常回答即可。

单次起卦对应单个问题。同一卦象的追问在本轮内继续解读，**不重摇**；新问题需用户重新触发。

**退出**：用户输入「收卦」或 `/exit-qigua` 时退出。注意 `/exit-qigua` 不是已注册的斜杠命令，纯文本「收卦」最可靠（一定送达模型）。退出后遵守三条：

1. **粘性**——本轮对话内卦象模式即休眠，后续不再触发，除非用户重新以 `/qigua` 或「起卦」开头提问。
2. **清玄学**——之后的回答彻底不出现卦象语言：不提卦名、爻、本卦/变卦、宜进宜缓、五行方位、"弃卦从实"等，像一个从没起过卦的普通助手在答。
3. **留结论**——已得出的思考方向与结论照常保留，用平实的话重述；尚无结论的，以纯事实分析补完。

## 核心原则

卦象的作用不是预测未来，而是**在你形成默认判断之前，强制注入一个非显然的思考切入角**。它能起作用的前提是真的随机、真的不可挑选。因此：

1. **真随机**：必须用 `Get-Random` 摇卦，禁止模型自行"心算"或挑选卦象。
2. **不二筮**：一事一卦，摇出什么解什么。"初筮告，再三渎，渎则不告"（《蒙》卦辞）——不许因为卦不顺眼而重摇。
3. **事实优先**：卦象提供视角，结论必须有事实支撑。推演结果与事实、数据、工程判断冲突时，事实定夺，并明说"此处弃卦从实"。

## 起卦五步

### 第一步：摇卦（必须执行命令）

```powershell
1..6 | ForEach-Object { $c = 1..3 | ForEach-Object { Get-Random -Minimum 2 -Maximum 4 }; "第{0}爻: {1} = {2}" -f $_, ($c -join '+'), (($c | Measure-Object -Sum).Sum) }
```

三枚铜钱：背=3（阳面），字=2（阴面）。第1爻为初爻（最下），自下而上排卦。

| 三钱之和 | 爻 | 记法 |
|---|---|---|
| 6 | 老阴 ⚋ → 变阳 | `━ ━ ×`（变爻） |
| 7 | 少阳 ⚊ | `━━━` |
| 8 | 少阴 ⚋ | `━ ━` |
| 9 | 老阳 ⚊ → 变阴 | `━━━ ○`（变爻） |

**断卦语义**：本卦=当前局面（事之始），变卦=做下去的演变走向（事之终），变爻=成败的关键变化点。无变爻则局面稳定，本卦即结论。

### 第三步：取辞（按变爻数定读法，朱子简则）

| 变爻数 | 读什么 |
|---|---|
| 0 | 本卦卦辞 |
| 1 | 该变爻爻辞 |
| 2 | 两变爻爻辞，以上者为主 |
| 3 | 本卦+变卦卦辞，本卦为体 |
| 4 | 变卦中两不变爻爻辞，以下者为主 |
| 5 | 变卦中唯一不变爻爻辞 |
| 6 | 乾读"用九"、坤读"用六"，余卦读变卦卦辞 |

引用爻辞/卦辞原文，附一句白话。凭记忆引文没把握时宁可只取卦义大方向，不得编造原文。

**取辞后必须提炼倾向**，落到四档之一，不许含糊：

- **宜进**——断辞偏吉（吉、亨、利、无咎），走向向好，可以开始干
- **宜缓**——方向不错但时机未到或根基未立，先做准备再动
- **宜止**——断辞偏凶（凶、悔、吝、勿用），此路代价大，不开始或先停
- **宜变向**——事可做但方式要换，本卦到变卦的演变指出了新路径

并用一句话讲清走向：本卦（现状）→ 变卦（做下去的趋势）意味着什么。

**断力度（体用生克·《梅花易数》体用总诀）**——给"倾向"一个机械、犀利的强弱依据，破除温吞套话：

- **定体用**：含变爻的经卦为**用**（所谋之事/对象），不含变爻的经卦为**体**（你自己）。变爻横跨上下两卦或无变爻时，此法不适用，退回卦爻辞，不硬套。
- **八卦五行**：乾兑金·震巽木·坤艮土·离火·坎水。
- **生克定吉凶**（口诀"用生体为吉，体克用无妨"）：

| 体用关系 | 一句判语 |
|---|---|
| 用生体 | 大吉——有外力来助，顺势可进 |
| 体克用 | 吉——你掌控此事，略费力而无妨 |
| 体用比和 | 顺——谋事顺遂，无大碍 |
| 体生用 | 耗——你在倒贴心力，费而难得 |
| 用克体 | 凶——此事反压制你，受制于外，慎动 |

- **看旺衰**：体卦当令则强（春震巽·夏离·秋乾兑·冬坎·四季月坤艮），失令则弱。体强能扛克，体弱受生亦难起——旺衰叠加在生克上，得出"这事对你是顺是逆、力道多大"的定性判语。
- 体用生克与"倾向四档"矛盾时，取更保守者作提示，**最终仍由第五步事实核证定夺**。

### 第四步：推演（卦象 → 可核证假设）

**先解象，再落地（高岛吞象"象数义理并重"法）**：禁止从卦名直接跳结论。先把卦象榨干三层，每层都追问"这映射到所问之事的什么处境"：

1. **象**——上下卦物象与互卦（离=火/光明/网罗、兑=泽/口舌/缺口、坎=水/险陷、艮=山/阻止…），卦的结构在描述一个什么局面；
2. **义**——卦辞爻辞的义理，指向什么行动姿态（该进该守、宜独宜众）；
3. **位**——变爻在第几位定阶段：初=起步、二三四=过程、五=主事得位、上=终局或物极必反。

三层解完，再结合下表象数，提出 2–3 个针对所问之事的**具体假设**。每条必须满足两个条件：

1. **具体到可检验**——"西部渠道更易出货""年轻女性客群转化更高"，而不是"宜往吉方"这种没法查证的套话；
2. **当场写明核证方法**——用什么数据、资料、代码或实验可以佐证或证伪它。

其中至少一条是证伪点："如果这个方向是错的，最先会在哪里露出破绽。"

**八卦象数速查（说卦传·后天）：**

| 卦 | 方位 | 五行 | 人物 | 时节 |
|---|---|---|---|---|
| 乾 | 西北 | 金 | 老父·长者·权威 | 秋冬之交 |
| 兑 | 西 | 金 | 少女·年轻女性 | 秋 |
| 离 | 南 | 火 | 中女·中年女性 | 夏 |
| 震 | 东 | 木 | 长男·青壮年男性 | 春 |
| 巽 | 东南 | 木 | 长女·青年女性 | 春夏之交 |
| 坎 | 北 | 水 | 中男·中年男性 | 冬 |
| 艮 | 东北 | 土 | 少男·少年 | 冬春之交 |
| 坤 | 西南 | 土 | 老母·中老年女性·大众 | 夏秋之交 |

象数维度（方位、人群、时节）适合市场、选址、人事、客群类问题；纯技术问题多取卦义层面，不硬套方位。

提出假设后**立即逐条核证**——查数据、查资料、查代码，该用什么工具用什么工具。核证必须**双向**：找佐证的同时必须找反例，不许只挑支持卦象的材料。

### 第五步：决断

**第一句必须是不超过一行的犀利定论**：做 / 不做 / 改成怎么做——直给，不铺垫、不"一方面另一方面"。条件、标注、风险一律放在定论之后。

随后：只采纳**核证通过**的假设；被证伪的假设明说"弃卦从实"并弃用，不许降格保留。区分标注哪些判断来自事实核证（定夺依据）、卦象起了什么作用（提出假设、换角度）。实事求是是最低底线——卦象负责出题，事实负责判卷。

## 输出格式

```
## 卜：[所问之事]

摇卦：[命令输出六行]
卦象：[六爻图，自上而下画] → 本卦 X（上X下X），变爻 [某爻]，之卦 Y
断辞：[原文] ——[一句白话]
断力度：体[X卦·Y行]·用[X卦·Y行] → [生克关系] → [一句判语]（不适用则写"—"）
倾向：[宜进/宜缓/宜止/宜变向]｜走向：[本卦→变卦一句话]
解象：象[…]·义[…]·位[…]（三层各一句）
假设：① [具体假设] ←核证法：[数据/资料/实验] ② …… ③ ……（含至少一个证伪点）

[逐条核证：佐证与反例并列，给出每条假设的核证结果]

## 决断
【一句话定论：做/不做/怎么做】
[随后：采纳哪些核证通过的假设、弃卦从实者、事实与卦象各自的作用]
```

## 一个例子（压缩）

问"要不要现在就重构模块 X"，摇得**水雷屯**之**水地比**（初九变）。
断辞：初九"磐桓，利居贞，利建侯"——徘徊不前不是坏事，宜先立根基、立帮手。
倾向：宜缓｜走向：屯（草创多难）→ 比（众人亲附），先立根基则人和事顺，方向可行但不急于今日动工。
假设：① 屯=草创多难：动工依赖未摸清 ←核证法：画出模块依赖图，数清入边；② "利建侯"：测试契约这个"侯"还没立 ←核证法：查该模块测试覆盖率；③ 证伪点：若下游使用方不愿配合迁移（比道不成），"现在重构"方向本身就错 ←核证法：问一遍三个下游团队的排期。
逐条核证后①②属实、③下游愿意配合，得出"先补集成测试再分两步重构"——结论由核证定夺，卦象的作用是把注意力从"怎么改"拉到了"谁依赖我"。

## 红线

- 不重摇、不挑卦、不心算出卦。命令失败就修命令，不许"我来模拟随机"。
- 核证不许单向：只找佐证不找反例＝确认偏误，等同渎卦。查无实据时如实说"查无实据"，不许用弱证据硬撑卦象。
- 卦象不得作为医疗、具体投资标的买卖、法律行动的依据；这些领域卦象至多供心态参考，结论须完全由事实与专业判断支撑。
- 用户若问"这准吗"，如实回答：这是结构化的换角度思考工具，随机性是用来打破思维定势的，不是预言。
